Building AI Agents Once and Deploying Them Everywhere

An introduction to the Chat SDK, a unified API for creating interactive AI agents that work across platforms like Slack, Discord, GitHub, and more.

The Challenge of Cross-Platform AI Agents

In early January, Vercel issued a company-wide mandate: figure out how to multiply your output. The response was overwhelming, with nearly every team building AI agents to handle complex workflows. Most of these were Slack bots designed for easy access within the company.

However, a significant problem emerged. Each time someone built an agent, they had to navigate the complexities of a specific platform's API. Integrating with Slack, for example, is harder than it seems. It requires understanding threads, reactions, how to post messages, and how to handle state. When another team member requested to use the same agent on GitHub, the process started all over again, duplicating logic for a different platform.

While chat APIs seem similar on the surface, they are completely different under the hood.

  • Slack supports native streaming, allowing tokens to be streamed directly into a message as an LLM generates them.
  • Discord requires a "post, edit, post, edit" method to simulate streaming.
  • GitHub offers no streaming at all.
  • Slack supports modals, while Discord does not.
  • Microsoft Teams provides read-only reactions.

These aren't minor quirks; they are fundamental differences in how each platform operates. A simple agent ends up with a huge amount of platform-specific logic, creating a maintenance nightmare. This isn't just a Vercel problem—every company will face the challenge of delivering AI agents to the surfaces where their teams already work, including chat, code review, and issue tracking.

The Solution: A Unified API

The AI SDK solved a similar problem for model providers, allowing developers to write code once that works across different models like GPT, Claude, or Gemini.

The Chat SDK does the same for interactive agents. It provides a single API for building agent interactions that work across multiple platforms, including Slack, GitHub, Linear, Discord, Telegram, and WhatsApp. You build the agent once, and the Chat SDK delivers it to users in the apps they already use.

Building V0 with the Chat SDK

Fernando, a developer at Vercel, shared his experience building V0 background agents. His goal was to create a general-purpose coding agent that could be tagged from Slack to open pull requests and run in any codebase.

After sharing the first version with friends, the immediate feedback was a request for support in Linear issues and GitHub PR comments. It became clear that the agent had to work everywhere. This led to a choice: build the agent platform by platform or build it once with a unified API. Having spent years building cross-platform apps with React Native, Fernando recognized the problem and turned to the Chat SDK. It allowed him to focus on building the core agent functionality without worrying about each platform's unique APIs.

How It Works: A Technical Demo

Matt from the Vercel team demonstrated how easy it is to get started.

Getting Started: "Hello, World!"

The initial example shows a simple bot that responds with "Hello, World!" when mentioned in Slack.

// A simple bot that responds on mention
createChat({
  // ... adapters
  listeners: {
    onMention: async ({thread}) => {
      thread.post('Hello, World!');
    }
  }
})

The code is straightforward: create a new chat instance, add an onMention listener, and post a message back to the thread.

Building Rich Native Experiences with JSX

To move beyond plain text, the Chat SDK integrates JSX, allowing developers to build rich, native UI components. Instead of a simple text response, you can create interactive cards.

// Using JSX to build a UI card
thread.post(
  <Card>
    <Text>Hello, World!</Text>
    <Button action="continue">Continue</Button>
    <Button action="cancel">Cancel</Button>
  </Card>
);

// Handling the button action
chat.onAction('continue', async ({ user }) => {
  console.log(`${user.fullName} clicked continue`);
});

When this code runs, the components render natively in Slack. Clicking the "Continue" button triggers the action handler instantly.

Expanding to New Platforms

The real power of the Chat SDK becomes apparent when adding support for new platforms. To bring the exact same experience to Discord, you only need to add the Discord adapter. No other code changes are required.

createChat({
  adapters: [
    new SlackAdapter(),
    new DiscordAdapter(), // <-- Add the Discord adapter
  ],
  // ...listeners
})

The same UI renders natively in Discord.

Streaming for AI Agents

Modern AI agents require streaming to provide a responsive user experience. With the Chat SDK, streaming is trivial. You create a stream using the AI SDK and post it to the thread with a single API call, without needing to know how each platform handles streaming or partial updates.

// Streaming a response from an AI agent
onMention: async ({thread}) => {
  const stream = createStream(...) // from AI SDK
  thread.post(stream);
}

When mentioned in Slack, the bot streams back its response natively.

Supporting Direct Messages and More

The Chat SDK isn't limited to mentions in channels. You can easily add support for direct messages on platforms like WhatsApp or Telegram by adding an onDirectMessage listener and the necessary adapters.

The same principle applies to other surfaces. By adding the GitHub adapter, the same agent that was available on Slack, Discord, and WhatsApp can now review code and respond to comments in pull requests.

Conclusion

The demonstration showcased how a single agent, built with a few lines of code in one file, was deployed across Slack, Discord, WhatsApp, Telegram, and GitHub. The Chat SDK abstracts away the complexities of each platform's unique API, allowing you to build rich, AI-powered experiences once and deploy them everywhere.

To get started, check out the documentation and templates at chat-sdk.dev.
