The Future is Local: How Claude Cowork Turns Your Mac into an AI Teammate

In a recent Latent Space podcast, host Alessio, founder of Kernel Labs, and Swixs, editor of Latent Space, were joined by Felix, a technical lead at Anthropic, to discuss the philosophy and architecture behind Claude Cowork. They explored why giving an AI agent its own sandboxed computer on your local machine is a game-changer for productivity and how this approach differs from purely cloud-based AI assistants.

What is Claude Cowork?

Felix introduced Claude Cowork as a user-friendly version of Claude Code, Anthropic's terminal-based coding assistant. However, "user-friendly" in this context doesn't mean less powerful. The core idea was to bring the agentic capabilities of Claude Code to a broader audience, including those not comfortable with the command line.

Cowork runs Claude Code within a Virtual Machine (VM) on the user's computer. This provides a safe, sandboxed environment with guardrails, making it convenient for people who don't want to open a terminal to get work done. It's designed to handle tasks beyond coding, such as managing expenses, organizing a knowledge base, or filling out receipts.

Why "User-Friendly" Can Be More Powerful

An interesting parallel was drawn to the evolution of Visual Studio Code. Initially perceived as a "dumbed-down" version of the full Visual Studio, VS Code ultimately became dominant due to its hackability and extensibility. Users could easily hook it into any workflow and build extensions for it.

Similarly, Claude Cowork is a superset of Claude Code's capabilities. Its convenience and deep integration with tools like Claude in Chrome make it exceptionally powerful. The value isn't just in the base model but in how easily users can map the tool onto their specific jobs and extend its functionality.

Building Products When Execution is Cheap

Anthropic's product development has shifted in the new AI paradigm. Where previously a product manager might spend weeks on customer research before drafting a spec, the current philosophy is "let's build all the candidates."

"We internally had Anthropic up now probably much closer to the point we're like don't even write a memo just like build like let's build all the candidates very quickly like let's just build all of them and then pick the best ones."

Execution has become so inexpensive that the team can create multiple prototypes to address a problem and then choose the most effective solution. The 10-day development time for Cowork was possible because it leveraged many pre-existing components and prototypes developed over the last year and a half.

The Unsung Hero: Your Local Computer

A core, contrarian thesis behind Cowork is the immense value of the local computer. Felix argued that Silicon Valley generally undervalues the local machine, yet most power users still rely on MacBooks over Chromebooks or iPads.

For an AI entity to be "tremendously useful," it needs access to the same tools the user has. A purely cloud-based agent is often hamstrung, requiring complex permissions and OAuth grants for every tool. By running locally, Cowork can more seamlessly interact with a user's files and environment. This led to a key architectural decision:

  • The VM as Claude's Personal Computer: Cowork runs in a lightweight Linux VM. This is the most powerful part of its architecture, providing safety, security, and the freedom for Claude to install necessary tools like Python or Node.js without interfering with the host system. It's analogous to giving a human employee their own computer to work on.
  • Deep Integration: Cowork is tightly integrated with Claude in Chrome. This allows it to interact with websites and web apps without needing dozens of brittle API connectors, which Felix noted many people are giving up on.
  • System Prompt Steering: Much of Cowork's specialized behavior for knowledge work comes from carefully crafted system prompts that guide the model on how to best use its tools for a given task.

From Manual Clicks to Automated Skills

The conversation highlighted several powerful workflows that demonstrate the progression from manual work to full automation.

Demo: Automating YouTube Uploads

Swixs showcased how he uses Cowork to manage the Latent Space YouTube channel.

  1. Manual Start: Initially, he manually downloaded recordings from Zoom and uploaded them to YouTube, a laborious, click-heavy process.
  2. Task Automation: He then instructed Cowork to perform the downloads and uploads.
  3. Intelligent Automation: The process became more advanced, with Cowork analyzing video frames to automatically generate relevant titles for the uploads.
  4. Self-Generated Skills: Finally, he asked Cowork to wrap the entire repetitive workflow into a skill. Cowork generated the skill file itself, which can now be reused with a single command. He even had it split the master skill into smaller, modular skills for more granular control.

This progression—from manual task to automated workflow to reusable, agent-generated skill—is the key to unlocking Cowork's potential.

Other Use Cases:

  • Bug-Fixing: Felix described a workflow where Cowork scans internal crash reports, identifies fixable bugs, and then spins up other Claude instances to write the code to fix them.
  • Desktop Organization: A viral use case was "clean up your desktop." While simple, it's a relatable task that demonstrates the agent's ability to interact with the local file system to organize files thematically.
  • Tax Preparation: Felix mentioned using Cowork for his taxes, feeding it financial documents to handle the complex knowledge work involved.

The Future of Cowork: Multiplayer and Portability

The discussion also touched on what's next for agentic workflows.

Skills Portability

A major topic was the portability of skills. Since skills are just markdown files, they are inherently portable. Anthropic uses a container format called plugins, which are GitHub repos containing collections of skills, available for both Cowork and Claude Code.

However, a challenge remains in making skills both portable and deeply personal. A "book a flight" skill, for example, has universal logic (find the cheapest flight) but also personal preferences (seat choice, preferred airports). The industry has yet to solve how to best structure these private and public components.

Multiplayer Agents

What does a "multiplayer mode" for Cowork look like? The hosts envisioned a future where agents could collaborate.

  • An agent could ask a colleague's agent for a file or a status update.
  • Agents could share skills with each other automatically. One idea floated was using Bluetooth to detect nearby coworkers and share relevant skills for a common task.

The path to this future could involve building custom communication protocols, or it could be as simple as giving each agent its own Slack handle and email address, allowing them to communicate using the same tools humans do.

Why Shipping Chromium with Electron Still Matters

Felix, who was instrumental in the creation of Electron, provided insight into why shipping a full version of the Chromium browser engine is crucial for desktop apps like Cowork and Slack.

While using an operating system's native web view seems more efficient, it's a trap. OS web views are often buggy and are only updated with major OS releases. If an app-breaking rendering bug exists, the only recourse is to tell customers to buy a new computer, which is unacceptable.

Chromium, in contrast, is a "marvel of engineering." It provides a stable, reliable, and constantly updated rendering engine that works across countless hardware and software configurations. Electron's magic lies in making it easy to package and ship Chromium, giving developers the control they need to deliver a reliable experience to all users.
